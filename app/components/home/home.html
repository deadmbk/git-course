Git

Jest to szybki, rozproszony system kontroli wersji dostępny na licencji open-source. Stworzył go i zaprojektował Linus Torvalds jako narzędzie wspomagające rozwój jądra Linux. Git stanowi wolne oprogramowanie i został opublikowany na licencji GNU GPL w wersji 2.

Dla Git-a inspiracją były narzędzia "Bitkeeper" i "Monotone". Początkowo był zaprojektowany jako narzędzie kontroli wersji niskiego poziomu, na którym inni mogli pisać swoje własne aplikacje. Jednak z biegiem czasu Git stał się w pełni rozwiniętym systemem kontroli wersji.

Głównymi cechami systemu są:

  • Mocne wsparcie dla nieliniowego tworzenia aplikacji- Git wspiera branch-owanie i merge-owanie oraz posiada narzędzia do wizualizacji i poruszania się po nieliniowej historii projektu.
  • Kompatybilnośc z istnijącymi systemami/protokołami- Dane można wymieniać przez HTTP(S), FTP, rsync, SSH
  • Rozproszone środowisko- Każdy programista posiada własną kopię repozytorium, do której może zapisywać zmiany bez połączenia z siecią; następnie zmiany mogą być wymieniane między lokalnymi repozytoriami.
  • Wydajna obsługa dużych projektów- System Git według zapewnień Torvaldsa, a także według testów fundacji Mozilla, jest o rzędy wielkości szybszy niż niektóre konkurencyjne rozwiązania.
  • Rewizja to obraz całego projektu- W przeciwieństwie do innych systemów kontroli wersji, Git nie zapamiętuje zmian między kolejnymi rewizjami, lecz kompletne obrazy.
